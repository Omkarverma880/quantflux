S5 Gann Range Entry: MARKET

Spot-index breakout from a floating Gann range with optional retest hold. One trade per day (default).

Entry trigger: Spot crosses the live upper_gann (→ BUY CE) or lower_gann (→ BUY PE) of the floating range. With retest_only = true (default), a retest hold is required.

Entry

MARKET BUY ATM/ITM option
Strike chosen at trigger time. No slippage cap — exchange fills at touch.

Stop Loss

fill − sl_points
shadow→ promotes to SL-M when ltp ≤ sl_price + sl_proximity.

Target

fill + target_points
or n-th Gann above fill if gann_target = true
shadow→ promotes to LIMIT on proximity.
ParamDefaultDescription
sl_points 30 Stop-loss distance (premium ₹).
target_points 60 Target distance when gann_target=false.
gann_target falseUse n-th Gann level instead of flat target_points.
gann_count 1 Number of Gann levels to target.
sl_proximity 5 Distance at which shadow SL goes live on exchange.
target_proximity 5 Distance at which shadow target goes live.
retest_only trueRequire retest hold; suppresses fake-out reversals.
allow_reentry falseOne trade per day when false.
max_entry_slippage 8 Premium slippage cap after fill; if breached, position is flattened immediately.
S5 still uses MARKET entry and the post-fill _slippage_flatten guard. Unlike S6–S9 it has not yet been converted to LIMIT-at-line entry. The flattening branch will reject the position if fill slippage exceeds max_entry_slippage.

S6 CALL / PUT Lines Entry: LIMIT @ option LTP

User-drawn spot lines. When spot crosses a line, buy the corresponding option (CE for CALL line, PE for PUT line).

Entry trigger: Spot crosses call_line (→ BUY CE) or put_line (→ BUY PE). Optional itm_offset shifts the strike deeper ITM.

Entry

LIMIT BUY @ option_ltp
Limit pegged at the option's LTP at trigger time. Gap-up moves rest unfilled instead of overpaying via MARKET. Fallback to MARKET only if LTP ≤ 0.

Stop Loss

fill − sl_points
shadow→ promotes to SL-M on proximity.
No more “if past SL: MARKET sell” fallback — broker SL-M handles all crossings cleanly.

Target

fill + target_points
shadow→ promotes to LIMIT on proximity.
ParamDefaultDescription
call_line user-drawnSpot level that, when crossed up, triggers BUY CE.
put_line user-drawnSpot level that, when crossed down, triggers BUY PE.
itm_offset 0 Strike offset (₹) for deeper-ITM entries.
sl_points 30 Stop-loss distance in option premium.
target_points 60 Target distance in option premium.
sl_proximity 5 Shadow-SL → exchange-SL trigger distance.
target_proximity5 Shadow-target → exchange-LIMIT trigger distance.
Slippage logic removed. LIMIT entry caps the fill at the exchange level — a runaway move rests unfilled rather than overpaying.

S7 Strike Lines Entry: LIMIT @ option LTP

User-drawn lines on option LTP charts (CE and PE separately). Symmetric — both legs can fire in one day.

Entry trigger: CE LTP crosses ce_line → BUY that CE strike. PE LTP crosses pe_line → BUY that PE strike. Lines are anchored to a previous tick (_prev_ce / _prev_pe) so editing lines on a live chart never causes false fires.

Entry

LIMIT BUY @ option_ltp
Same LIMIT-at-line semantics as S6. The line itself is on option LTP, so option_ltp ≈ line at trigger.

Stop Loss

fill − sl_points
shadow→ SL-M on proximity. Clean promotion — no breach-MARKET branch.

Target

fill + target_points
shadow→ LIMIT on proximity.
ParamDefaultDescription
ce_line user-drawnCE LTP trigger.
pe_line user-drawnPE LTP trigger.
sl_points 30 Stop-loss distance.
target_points 60 Target distance.
sl_proximity 5 Shadow→exchange SL promotion distance.
target_proximity 5 Shadow→exchange target promotion distance.
max_trades_per_day2 Cap on entries per session (one CE + one PE typically).
LIMIT entry replaced MARKET. The legacy max_entry_slippage param still exists for backward compatibility but the post-fill flatten branch is removed.

S8 Reverse Lines Entry: LIMIT @ option LTP

User draws lines on CE / PE LTP; the trigger buys the opposite side (CE touch → BUY PE, and vice-versa).

Entry trigger: CE LTP touches ce_line → BUY reverse PE. PE LTP touches pe_line → BUY reverse CE. The reverse strike is chosen by reverse_mode: AUTO uses a 200-pt ITM offset; MANUAL uses an operator-specified strike.

Entry

LIMIT BUY @ option_ltp (reverse-strike LTP)
LIMIT pegged at the LTP of the opposite option being bought, not the line being touched.

Stop Loss

fill − sl_points
shadow→ SL-M on proximity.

Target

fill + target_points
shadow→ LIMIT on proximity.
ParamDefaultDescription
ce_line / pe_line user-drawnLines drawn on CE/PE LTP.
reverse_mode AUTO AUTO uses reverse_offset; MANUAL uses fixed strikes.
reverse_offset 200 Strike offset (₹) for the opposite leg in AUTO mode.
manual_ce_strike — Manual override (MANUAL mode only).
manual_pe_strike — Manual override (MANUAL mode only).
sl_points 30 Stop-loss distance.
target_points 60 Target distance.
sl_proximity 5 Shadow→exchange SL distance.
target_proximity 5 Shadow→exchange target distance.
LIMIT entry + clean shadow promotion. No _slippage_flatten, no breach-MARKET fallback.

S9 Line Of Control (LOC) Entry: LIMIT @ buy line

Six user-drawn lines (CE buy / target / SL and PE buy / target / SL). Entry and exits are all anchored to user lines, not computed from fill. Lines can be dragged live and resting orders are auto-cancelled and re-promoted at the new price.

Entry trigger: CE LTP ≥ ce_buy_line → BUY CE.
PE LTP ≥ pe_buy_line → BUY PE.
Single-cycle lock: while one leg is open, the opposite leg cannot trigger.

Entry

LIMIT BUY @ buy_line (exact)
No slippage buffer. Exchange enforces fill ≤ buy_line for LIMIT BUY. Gap-up rests unfilled until the line is hit again or operator drags the line up.

Stop Loss

sl_line (user-drawn)
shadow→ SL-M @ sl_line when ltp ≤ sl_line + sl_proximity.
If line is dragged while resting order exists → resting order is cancelled and re-promoted at new line on the next tick.
MARKET fallback only if hard-breached while still in shadow.

Target

target_line (user-drawn)
shadow→ LIMIT @ target_line on proximity. Live-drag handling same as SL.
ParamDefaultDescription
ce_buy_line / pe_buy_line user-drawnLIMIT entry price for each side.
ce_target_line / pe_target_line user-drawnProfit-take line per side.
ce_sl_line / pe_sl_line user-drawnStop-loss line per side.
sl_proximity 5 Distance at which shadow SL is promoted to SL-M.
target_proximity 5 Distance at which shadow target is promoted to LIMIT.
stale_cancel_seconds 10 Cancel + re-promote idle resting orders to handle live line drags.
All six lines are editable live during a trade. The exit promotion logic detects drags > 0.04 ₹ and rolls the resting order without disturbing the position.

S1 Gann CV (Buy) Entry: LIMIT @ Gann

Cumulative-Volume signal → BUY ATM option at the next Gann support/resistance level.

Entry trigger: CV > cv_threshold → BUY ATM CE at floor Gann level. CV < -cv_threshold → BUY ATM PE at floor Gann level.

Entry

LIMIT BUY @ floor Gann level
Stored as gann_entry_price.

Stop Loss

entry − sl_points or prev Gann (if < sl_points)
Shadow → SL-M on proximity.

Target

entry + target_points or ceiling Gann (gann_target=true)
Shadow → LIMIT on proximity.

S2 Option Selling Entry: LIMIT SELL @ Gann

Inverse of S1 — sell premium at ceiling Gann level when CV signal fires.

Entry trigger: CV > threshold → SELL ATM PUT @ ceiling Gann. CV < -threshold → SELL ATM CALL @ ceiling Gann.

Entry

LIMIT SELL @ ceiling Gann level
Position is short premium.

Stop Loss

entry + sl_points (premium rises against us)
Shadow → SL-M BUY on proximity.

Target

entry − target_points or prev Gann
Shadow → LIMIT BUY on proximity.

S3 CV + VWAP + EMA200 + ADX Entry: LIMIT @ option LTP

Three-phase trend-pullback-breakout filter. The strictest entry gate of any strategy.

Entry trigger (all three must pass):
1. Trend align  — spot vs EMA200, VWAP, ADX ≥ threshold, |CV| ≥ threshold (when filter enabled).
2. Pullback  — price returns to EMA20.
3. Breakout candle  — close > open (bull) or close < open (bear) confirms.

Entry

LIMIT BUY @ option_ltp
ATM option in trend direction.

Stop Loss

fill − sl_points (with optional trailing)
Shadow → SL-M on proximity. Trail follows favourable moves.

Target

fill + target_points
Shadow → LIMIT on proximity, OR trend-flip exit when spot crosses EMA200 against direction.

S4 High-Low Retest Entry: MARKET

Yesterday's 9:15–10:15 high/low breakout with retest hold. Optional fake-out reversal entry.

Entry trigger: Spot breaks prior-day 9:15–10:15 high (→ CE) or low (→ PE), then retests and holds. Fake-out reversal logic can fire the opposite side if the breakout fails.

Entry

MARKET BUY option
Same MARKET-entry pattern as S5 (slippage-cap fallback active).

Stop Loss

fill − sl_points
Shadow → SL-M on proximity.

Target

fill + target_points or n-th Gann above fill
Shadow → LIMIT on proximity.
